# Word Count with Configurable Limits Implementation Plan

**Goal:** Activity and Course models support word limit configuration, resolved through PlacementContext.

**Architecture:** Three nullable fields on Activity (word_minimum, word_limit, word_limit_enforcement), one non-nullable default on Course (default_word_limit_enforcement). PlacementContext resolves enforcement via existing resolve_tristate(). Single Alembic migration.

**Tech Stack:** SQLModel, Alembic, PostgreSQL

**Scope:** 6 phases from original design (phase 2 of 6)

**Codebase verified:** 2026-03-02

---

## Acceptance Criteria Coverage

This phase implements and tests:

### word-count-limits-47.AC2: Data model and settings
- **word-count-limits-47.AC2.1 Success:** Activity `word_minimum` and `word_limit` accept positive integers or None
- **word-count-limits-47.AC2.2 Success:** Activity `word_limit_enforcement` is tri-state (None = inherit from course)
- **word-count-limits-47.AC2.3 Success:** Course `default_word_limit_enforcement` defaults to False (soft)
- **word-count-limits-47.AC2.4 Success:** PlacementContext resolves enforcement via existing `resolve_tristate()` pattern
- **word-count-limits-47.AC2.5 Failure:** Setting `word_minimum >= word_limit` (when both set) is rejected with validation error
- **word-count-limits-47.AC2.6 Edge:** Activity with no limits set -- no word count behaviour activated
